# Premium Report Expansion Design (2026-03-09)

## 1. Goal
NameFit 프리미엄 결과를 "TOP 5 집중형 리포트"로 개편한다. 핵심은 추천 개수 축소(20 -> 5), 카드별 상세 해석 강화, 공유 가능한 리포트 이미지 제공이다. 구현 범위는 web/toss 공통이며, 계산 엔진 자체를 교체하지 않고 리포트 컴포저 레이어를 추가한다.

## 2. Final Decisions
- 프리미엄 결과 노출 수는 5개로 고정한다.
- 카드 확장 UX는 단일 아코디언으로 한다.
  - TOP1만 기본 펼침
  - TOP2~TOP5는 기본 접힘
  - 카드 탭 시 한 번에 1개만 펼침
- 카드 액션 버튼은 2단 배치로 고정한다.
  - 1행: 좋아요 | 싫어요
  - 2행: 공유하기(단독 전체 폭)
- 공유는 "리포트 전용 이미지"를 생성해 OS 공유 시트를 호출한다.
  - 프리미엄: 상세 리포트형 이미지
  - 무료: 현재 카드 형태와 동일한 축약형 이미지
- 무료모드 내 유료모드 안내는 "곧 오픈" 톤을 유지하되, 실제 현재 범위 기준 문구만 사용한다.
- 연령대 해석은 5구간(0~19, 20~39, 40~59, 60~79, 80~100), 구간당 2문장으로 고정한다.
- 문장 톤은 부드러운 상담형 + 토스 톤앤매너로 통일한다.
- 문장 라이브러리는 v1에서 총 240개를 구축한다.

## 3. Premium Result UX Spec
### 3.1 Card Information Layers
- Collapsed(요약):
  - 이름/한자/점수/TOP 배지
  - 태그(오행/인상)
  - 핵심 근거 2~4개 짧은 불릿
- Expanded(상세):
  - 긴 해석 문단(요약 서사)
  - 연령대 5구간 해석(구간당 2문장)
  - 필요 시 주의 포인트 블록

### 3.2 Interaction
- expandedCardId 단일 상태로 제어
- 초기값은 TOP1 카드 id
- 접힘 카드 선택 시 기존 펼침은 즉시 닫힘
- 좋아요/싫어요/공유는 카드 단위 pending 상태 분리

### 3.3 Share UX
- 카드 DOM 캡처가 아닌 Share 전용 템플릿 렌더 후 PNG 생성
- 프리미엄 공유 이미지 포함 요소:
  - 이름/한자/점수/핵심 태그
  - 핵심 근거
  - 연령대 5구간 요약
  - 브랜드 라벨
- 무료 공유 이미지 포함 요소:
  - 현재 무료 결과 카드와 동일한 카드 프레임
  - 좌상단: 성 포함 전체 이름
  - 하단 중앙: 한자 2개 블록
  - 각 한자 블록 정보 순서: 현재 화면과 동일(한자 -> 음 -> 뜻)
- 제외 요소:
  - 버튼, 토스트, 입력/디버그 UI
  - 무료 공유에서는 점수/추천 이유/추가 문장 미포함
- 전송 순서:
  - navigator.canShare(files) 가능 -> navigator.share(files)
  - 불가 시 다운로드 fallback

## 4. Report Phrase System (240)
### 4.1 Library Composition
- 공통 톤 문장: 30
- 연령대 슬롯 문장: 150
  - 5구간 x 3슬롯(흐름/보완/주의) x 10변형
- 조건 보정 문장: 40
  - 부족/과중 오행, 음양 비율, 일간 조건
- 연결/완충 문장: 20
- 총 240

### 4.2 Output Rule
- 연령대 최종 출력은 구간당 2문장
- 슬롯 후보는 조건 기반 필터링 후 선택
- 같은 카드 내 반복어 최소화(토큰 중복 제한)
- 같은 세션 내 카드 간 문장 ID 재사용 제한

## 5. Architecture
### 5.1 Keep Existing Engine
기존 사주 계산/추천 스코어 로직은 유지한다. 리포트 품질 개선은 별도 컴포저 계층으로 분리한다.

### 5.2 New Modules
- reportPhraseCatalog.ts
  - 문장 240개 + 메타(슬롯 타입, 연령대, 조건 태그)
- reportRules.ts
  - 조건 필터(부족/과중 오행, 음양, 일간)
- premiumReportComposer.ts
  - 후보별 상세 리포트 최종 조합
- shareNameCardImage.ts
  - 공유 템플릿 렌더 + 이미지 생성 + 네이티브 공유/fallback

## 6. Free Mode Premium Teaser Copy Policy
"곧 오픈" 문구는 유지하되, 과장 없이 현재 구현 범위 기준으로만 표기한다.

권장 문구 구조:
- 제목: 유료 모드도 곧 오픈해요 ✨
- 기능 포인트:
  - 사주 기반 상위 5개 이름 추천
  - 후보별 상세 리포트(연령대 5구간)
  - 부족/과중 오행 중심 해설
- 보조문구는 사용하지 않는다.

## 7. QA Criteria
- 프리미엄 결과가 정확히 5개만 노출되는지
- TOP1 기본 펼침 + 단일 아코디언 동작이 일관적인지
- 좋아요/싫어요/공유 로딩 상태가 서로 간섭하지 않는지
- 공유 이미지에 버튼/토스트가 포함되지 않는지
- 무료 공유 이미지가 현재 카드 구조(이름 위치/한자 블록/정보 순서)와 동일한지
- 연령대 5구간 x 2문장 구조가 모든 카드에서 유지되는지
- 조건 불일치 문장(예: 부족 오행과 반대 표현)이 출력되지 않는지
- 무료모드 유료 안내 문구가 정책과 일치하는지

## 8. Rollout Proposal
- Phase 1: 데이터/컴포저(문장 라이브러리 + 생성 규칙)
- Phase 2: 프리미엄 카드 UI(5개/아코디언/2단 버튼)
- Phase 3: 공유 이미지 + OS 공유 + fallback
- Phase 4: 카피 정리 + QA + 릴리즈
